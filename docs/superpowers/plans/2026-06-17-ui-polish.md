# UI Polish: Tabbed Detail Pages & Fixes — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Restructure lure (and retrofit species/brand) detail pages into path-addressable tabs, surface variant images, and apply eight smaller UI fixes (footer translations, copyright removal, comment spacing, upvote visual state, lure-index hero).

**Architecture:** Server-rendered ERB + Hotwire. Tabs become real URLs via an optional whitelisted `:tab` path segment served by each resource's `show` action; a shared `shared/_tabs` partial renders the tab bar as links. The `tabs_controller.js` Stimulus controller is retained solely for the `/design-system` styleguide.

**Tech Stack:** Rails 8.1, SQLite, Hotwire, Minitest integration tests, I18n (maintained locales: en, de, bg, ja).

## Notes for the implementer

- **Route ordering is load-bearing.** For each resource, declare `resources ... only: %i[index new create edit]` BEFORE the custom `get ".../:id(/:tab)"` so `/x/new` and `/x/:id/edit` match their actions, not the catch-all show. The `constraints: { tab: ... }` whitelist keeps `/edit` out of the tab matcher. Always run `bin/rails routes -g <name>` after editing and confirm the helper resolves to the GET show route.
- **`tabs_controller.js` stays.** It is still used by `/design-system`. Do not delete it; do not add `data-controller="tabs"` to the converted content pages.
- **Tab data is always loaded.** Controllers compute every tab's data each request (cheap, unchanged from today). Only the rendered panel switches on `@tab`. This keeps the diff small and avoids per-tab query branching.
- **Default tabs:** lures→"caught", species→"lures", brands→"lures". The default has no path segment (`/lures/:slug`).
- **Locales:** only en/de/bg/ja. The parity test enforces de/bg/ja; keep keys identical, translate values, preserve any `%{...}` placeholders.
- **Existing patterns to mirror:** `shared/_provenance`, `shared/_leaderboard_table`, `shared/_catch_card`, `shared/_lure_card` partials; the `lb-table` card style; inline styles are the house norm in these views.
